# Codex 斜杠指令执行复杂任务完整实操方案（适配 codex-cli 0.142.0）
复杂任务定义：多模块项目重构、全仓库代码标准化、自动化流水线、多步骤工程交付、深度漏洞审计、前后端一体化脚手架开发等**多阶段、多文件、长链路**需求。
核心逻辑：**环境校验 → 上下文清理 → 模型算力分配 → 标准化技能约束 → Agent自主执行 → 结果校验**，依靠斜杠指令分层管控，避免单轮自然语言指令失控。

## 一、复杂任务通用执行标准流程（固定执行顺序，必按顺序）
### 步骤1：前置环境自检 `/status`
作用：确认当前权限、可读写目录、剩余上下文token、模型档位、沙箱/本地模式、工具挂载状态，提前规避权限不足、上下文溢出、工具未启用问题。
```bash
/status
```
读取输出重点：
- File Scope：确认需要修改的目录在读写白名单内；
- Context Remain：剩余token不足30%必须执行 `/compact`；
- Model：确认当前档位，复杂任务后续切换 `/model max`；
- Agent Permission：默认read-only，批量修改需手动切换edit。

### 步骤2：上下文净化，释放窗口
两种方案按需选择：
1. 迭代型项目（保留原有架构/规则）：`/compact`
   折叠冗余闲聊，保留代码、项目规则、自定义skills，不清除核心上下文；
2. 全新大型任务（无历史依赖）：`/clear`
   彻底清空全部历史，杜绝旧需求干扰新任务逻辑。
```bash
# 迭代改造用
/compact
# 全新从零搭建项目用
/clear
```

### 步骤3：分配推理算力 `/model`
复杂任务涉及架构设计、多层逻辑校验、跨文件联动，必须切换深度思考模式：
```bash
/model max
```
补充说明：
- `/model speed` 仅用于简单格式化，严禁复杂重构、漏洞排查；
- `/model balance` 中小型模块迭代可用，上万行级项目强制max；
- 档位仅当前会话生效，任务中途逻辑错乱重新执行 `/model max`。

### 步骤4：加载标准化约束 `/skills run`（复杂任务核心稳定手段）
提前把团队编码规范、项目架构约束、交付标准封装为Skill，执行任务前一键加载，不用每次重复写大段规则，统一AI输出标准。
```bash
# 示例：加载自定义工程规范技能
/skills run 企业后端代码规范审计
```
如果无预制技能，可现场创建临时技能锁定规则：
```bash
/skills create 复杂项目重构标准
规则：分层架构、单一职责、禁止深层嵌套、全量异常捕获、生成变更日志、不破坏原有接口
```

### 步骤5：启动自主智能体Agent，赋予执行权限
复杂任务手动分步效率极低，`/agent` 自动拆解任务、分阶段执行、自我校验BUG，支持数十轮连续操作。
权限分级按需选择（安全优先级：read-only > edit > terminal）
```bash
# 1. 安全预览（推荐首次执行）：只读扫描，仅输出报告，不修改文件
/agent read-only
# 2. 常规代码重构：允许新增/修改代码文件
/agent edit
# 3. 全工程自动化（慎用）：可执行npm、shell、安装依赖
/agent terminal
```

### 步骤6：定义终极目标 `/goal`（完整复杂任务描述）
`/goal` 是Agent的任务总纲领，必须写清**阶段拆分、交付物、约束条件、输出格式**，不能模糊描述。
#### 合格复杂goal模板结构
```
/goal
请分6阶段自主完成当前项目全套升级工程，全程分步执行，每完成一个阶段输出阶段总结：
阶段1：扫描全项目目录，梳理现有架构、模块依赖、文件清单；
阶段2：统一全部代码命名、注释、缩进规范，修复所有语法错误；
阶段3：重构嵌套冗余逻辑，抽离公共工具类，实现低耦合分层；
阶段4：遍历所有业务代码，排查边界漏洞、安全隐患、性能瓶颈；
阶段5：为所有核心函数生成单元测试用例，覆盖正常/异常场景；
阶段6：输出完整项目改造报告，包含全部修改文件、优化点、风险提示；
约束：不删除原有业务功能、不改动对外接口、所有修改生成变更日志便于核对。
```

### 步骤7：任务中途管控指令
1. 执行逻辑偏离预期、修改风险过高：`/stop` 立刻终止Agent循环；
2. 想查看每一步操作记录：`/log` 导出完整执行日志；
3. 上下文再次溢出卡顿：重新执行 `/compact`；

## 二、3类典型复杂任务完整指令组合案例（可直接复制使用）
### 案例1：大型存量项目全量重构（上万行代码，最高频复杂任务）
```
/status
/compact
/model max
/skills run 后端工程统一规范
/agent edit
/goal
自主分阶段重构当前src目录下全部业务代码：
1. 输出项目原始目录架构与依赖关系；
2. 统一变量、函数、文件夹语义化命名，清理废弃代码；
3. 拆分超长函数，分层controller/service/util架构；
4. 修复隐性逻辑BUG、空指针、参数未校验等风险；
5. 统一全局异常处理、日志打印格式；
6. 生成完整改造清单，标注每一处文件修改内容；
全程保留原有业务逻辑，不修改对外API出入参格式。
```

### 案例2：从零搭建前后端一体化脚手架（多文件、多目录、多配置）
```
/clear
/status
/model max
/cloud
/agent edit
/goal
从零搭建轻量化Vue3+Python FastAPI全栈脚手架，分步完成：
1. 生成标准前后端分离目录结构，说明每个文件夹作用；
2. 前端：创建vite配置、路由、组件封装、请求工具；
3. 后端：创建项目入口、配置文件、数据库模型、接口分层；
4. 生成requirements.txt、package.json、README部署文档；
5. 补充启动命令、环境变量模板、基础使用示例；
要求代码可直接运行，注释完整，适配Python3.10+、Node18+。
```

### 案例3：企业级代码安全审计（只读分析，无文件修改）
```
/status
/compact
/model max
/skills run 代码安全审计标准
/agent read-only
/goal
对当前项目全量代码做安全风险审计，禁止修改任何文件：
1. 扫描明文密钥、硬编码密码、内网地址；
2. 排查SQL注入、XSS、未过滤用户输入等安全漏洞；
3. 识别废弃高危函数、未做权限校验接口；
4. 按风险等级（高/中/低）输出标准化审计报告；
5. 针对每一处风险给出修复方案与修改示例。
```

## 三、执行复杂任务核心避坑要点 ⚠️
1. **禁止无Agent单轮下发超大需求**
   仅靠自然语言长文本描述复杂任务，AI不会自动拆分步骤，极易遗漏模块、逻辑残缺；必须搭配 `/agent + /goal` 自动分阶段执行。
2. **复杂任务不要频繁切换会话**
   Agent会记忆项目目录、文件结构、工程规则，中途新建会话会丢失上下文，需要重新梳理项目，浪费大量token。
3. **终端权限 `/agent terminal` 谨慎开启**
   允许AI自主执行删除、安装、覆盖文件命令，操作前务必备份项目源码，防止误删核心配置。
4. **超大项目拆分模块执行**
   超过500文件的仓库不要一次性全量处理，分「工具层→业务层→配置层」分批执行，每批次执行完 `/compact` 释放上下文。
5. **高消耗指令组合控制频次**
   `/model max + /agent` 组合token消耗极高，批量复杂任务建议间隔10分钟以上，避免触发OpenAI速率限制。
6. **修改类任务必须输出变更日志**
   在 `/goal` 强制要求输出修改清单，人工核对后再保存项目，防止隐性逻辑改动导致线上故障。

## 四、复杂任务失败快速修复方案
1. AI输出逻辑混乱、遗漏模块：执行 `/compact` 压缩上下文，重新运行 `/goal`；
2. 模型思考浅显、重构不彻底：重新执行 `/model max` 切换深度推理；
3. Agent无法修改文件：`/status` 查看读写权限，切换 `/agent edit`；
4. 任务执行一半强制中断：`/log` 查看已完成步骤，调整 `/goal` 从断点继续执行；
5. 输出代码格式不统一：提前 `/skills run` 加载编码规范技能锁定输出标准。
