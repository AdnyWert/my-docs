## 第1章 核心通识：Codex 斜杠指令体系总览 🧩

### 1\.1 什么是 Codex 斜杠指令？

斜杠指令（Slash Commands）是 Codex **原生内置的底层特权指令**，不属于自然语言对话，是直接调用 Codex 内核、调度模型、切换智能体、激活技能、修改会话权限的官方入口。

普通用户靠“说话”操控 AI，高级用户靠**斜杠指令**操控 AI 内核。

**核心区别**：

- 自然语言：模糊、依赖理解、不可控、易跑偏

- 斜杠指令：精准、内核级触发、无歧义、可标准化、可工程复用

### 1\.2 指令四大分类（本篇全覆盖）

1. **状态管控指令**：查看会话、额度、权限、环境信息

2. **模型调度指令 /model**：切换推理模型、调节算力、速度/精度档位

3. **智能体控制指令 /agent /goal**：开启自主执行、循环任务、无人值守工作流

4. **技能系统指令 /skills**：查看、触发、创建、封装自定义工程技能

### 1\.3 通用使用规则与注意事项 ⚠️

- 指令必须以 `/` 开头，无多余修饰、无需解释需求

- 部分指令**仅 Codex 专属**，普通 ChatGPT 对话窗口无效

- 指令执行优先级高于自然语言需求，触发后会优先执行内核逻辑

- 高频调度模型/Agent 会消耗更多 Token 与额度，大型任务建议分批执行

## 第2章 基础管控类斜杠指令（全详解\+案例）📊

本章为通用高频指令，用于日常会话运维、状态排查、环境校准，是高级使用的基础工具集。

### 2\.1 /status 会话状态查询【最高频】

**指令功能**：一键查看当前 Codex 会话完整运行状态，是排错、调参前的必备前置指令。

**返回信息包含**：当前使用模型、推理档位、文件读写权限、可操作目录、Token 消耗、上下文余量、审批策略、云环境状态。

**使用案例**：

```Plain Text
/status
```

**适用场景**：任务报错、模型不对劲、权限异常、额度不足、会话卡顿排查

**注意事项**：无任何副作用，可随时执行，建议复杂任务前必查。

### 2\.2 /compact 上下文压缩优化

**指令功能**：自动精简当前对话上下文，将冗余历史折叠为摘要，释放 Token 空间，解决上下文溢出、会话错乱、后续输出跑偏问题。

**使用案例**：

```Plain Text
/compact
```

**适用场景**：长会话迭代、大型项目开发、多轮修改后、输出逻辑混乱时

**注意事项**：压缩后**不会删除已有代码与核心规则**，仅精简冗余对话，安全无风险。

### 2\.3 /cloud /cloud\-environment 云端沙箱调度

**/cloud**：快速切换至云端沙箱运行模式，所有文件、命令、代码均在远程云环境执行，不占用本地设备资源。

**/cloud\-environment**：手动选择指定云端环境版本、运行容器、系统内核。

**使用案例**：

```Plain Text
/cloud
/cloud-environment
```

**适用场景**：本地环境缺失、依赖冲突、需要纯净运行环境、批量测试任务

**注意事项**：云端环境数据会话结束后会清空，重要文件需手动导出保存。

### 2\.4 /log 日志查询

**指令功能**：查看当前会话所有执行日志、修改记录、报错堆栈、命令执行记录。

**使用案例**：

```Plain Text
/log
```

**适用场景**：排查隐性报错、追溯代码修改记录、定位任务失败原因

### 2\.5 /feedback 问题反馈

**指令功能**：一键唤起官方反馈面板，自动携带当前会话日志，用于提交功能异常、BUG、能力建议。

## 第3章 /model 模型调度指令（高阶调参完全手册）⚙️

**/model** 是 Codex 高级玩家与普通用户的分水岭，用于**手动切换推理模型、调节思考强度、速度/精度优先级**，彻底解决“太笨、太慢、太随意”的问题。

### 3\.1 /model 核心功能定义

默认 Codex 会自动适配模型，**手动 /model 调度可强制锁定算力档位**，适配不同难度任务。

**可调维度**：模型版本、推理算力、思考时长、创造力、严谨度、任务偏向（速度/质量）。

### 3\.2 三档模型策略（官方标准档位\+落地场景）

#### 档位1：极速模式（Speed）

**特点**：响应快、算力低、适合简单重复任务

**适用**：代码格式化、注释补充、简单纠错、文件整理

#### 档位2：均衡模式（Balance）【默认】

**特点**：速度与精度平衡，日常开发通用

**适用**：常规代码生成、功能迭代、小型项目重构

#### 档位3：深度思考模式（Max）

**特点**：超长思考链路、多轮自我校验、极高严谨度、消耗额度高

**适用**：复杂算法、架构设计、大型重构、BUG深度排查、工程优化

### 3\.3 /model 实操完整案例

**案例场景**：需要进行大型项目架构重构，强制开启深度思考模型

**操作流程**：

```Plain Text
/model
```

弹出面板手动选择：**Max/深度推理档位**，锁定高精度模式，再下发重构指令。

### 3\.4 /model 使用禁忌与注意事项 ⚠️

- 简单任务不要开 Max 模式：浪费额度、响应变慢、过度冗余

- 复杂任务不要用极速模式：容易逻辑漏洞、重构不彻底、漏判边界

- 模型切换**仅当前会话生效**，新建会话重置为默认均衡模式

- 高频深度推理会触发速率限制，大型工程建议分模块隔日迭代

## 第4章 /agent 智能体指令（自主执行工作流核心）🤖

**/agent** 是 Codex 最强黑科技，区别于普通对话：**开启后 AI 拥有自主规划、自主分步、自主迭代、无人值守执行能力**，可运行数十轮、数小时持续完成复杂工程。

### 4\.1 /agent 指令核心定义

**功能本质**：激活 Codex 原生 Agent 智能体循环（Ralph Loop），脱离单轮问答限制，自动拆解任务、制定执行计划、分步落地、自我校验修复。

**能力上限**：支持长周期自动化工程、全项目迭代、全流程落地、自主排错优化。

### 4\.2 /agent 配套核心指令 /goal（终极自动化）

**/goal** 是 Agent 的触发入口，用于定义**终极目标**，让智能体自主跑完完整工程，无需人工干预。

### 4\.3 Agent 完整落地案例（可直接复刻）

**场景**：让 Codex 自主完成「老旧项目全套重构优化工程」

**标准执行流程指令组**：

```Plain Text
/agent
/goal 自主完成当前项目全套工程升级：1.梳理项目结构 2.重构所有冗余代码 3.统一编码规范 4.修复全部基础BUG 5.优化性能 6.生成项目优化报告，全程自主分步执行，完成后输出完整变更日志
```

**执行效果**：AI 自动拆分十几步任务，逐轮迭代、自我检查、自动修正，全程无需你干预。

### 4\.4 Agent 权限档位说明（安全核心）

- **只读模式**：仅分析、查看、输出报告，不修改任何文件，最安全

- **可编辑模式**：允许修改项目文件、重构代码（默认常用）

- **终端执行模式**：允许自主运行终端命令、安装依赖、执行脚本（高阶慎用）

### 4\.5 Agent 高阶避坑注意事项（必看）🔒

- 开启 Agent 前**务必备份项目**，智能体会自主批量修改文件

- 首次使用建议先开「只读模式」观察执行计划，再放开编辑权限

- 禁止给 Agent 无限制全盘权限，避免误删核心文件

- 超长任务可随时暂停，输入 `/stop` 终止智能体循环

- Agent 不适合极度不确定的需求，目标必须清晰、结果可落地

### 4\.6 /stop 终止指令（配套必备）

**指令功能**：强制停止当前 Agent 所有自主任务、终止模型推理、结束循环工作流。

```Plain Text
/stop
```

## 第5章 /skills 技能系统（自定义能力封装\+复用）🎯

**/skills** 是 Codex 工程复用的核心，可将「一套固定流程、一套规范、一套检查逻辑」封装为**自定义专属技能**，一次创建、永久复用、批量落地。

### 5\.1 /skills 核心能力定义

- 查看当前已拥有的所有内置/自定义技能

- 一键触发预设标准化工作流

- 自主创建专属工程技能

- 固化个人/团队编码规范、审查标准、迭代流程

### 5\.2 /skills 基础查询用法

**指令**：

```Plain Text
/skills
```

**返回内容**：所有可用技能列表、技能功能简介、触发方式、适用场景。

### 5\.3 实战：创建自定义 Skill（工程级复用案例）

**场景需求**：创建一个【前端代码标准化审查技能】，后续任意项目一键触发审查。

**创建指令**：

```Plain Text
请为我创建一个可复用的代码审查 Skill，命名为【前端标准化审查】
技能固定规则：
1. 检查组件命名是否符合语义化规范
2. 检查是否引入冗余依赖、无效导入
3. 检查是否存在未定义变量、隐性语法风险
4. 检查代码嵌套层级是否超标
5. 输出标准化审查报告 + 对应修复建议
后续我调用该技能时，自动对当前项目代码执行全量审查
```

### 5\.4 触发自定义 Skill 案例

技能创建完成后，后续任意会话一键调用：

```Plain Text
调用技能：前端标准化审查，对当前src目录全量扫描审查
```

### 5\.5 高阶 Skill 落地场景

- 代码规范自检技能（团队统一标准）

- 项目打包优化技能（固定优化流程）

- BUG 自动体检技能（批量排查常规隐患）

- 文档自动同步技能（代码更新自动更新说明）

### 5\.6 Skills 使用注意事项 ⚠️

- 技能创建后**仅当前账号永久生效**，跨设备同步无需重复创建

- 技能规则尽量写死、标准化，避免模糊描述导致复用不稳定

- 复杂技能建议拆分多个子技能，单一职责、便于迭代更新

- 定期更新技能规则，适配最新编码规范与项目架构

## 第8章 终极避坑总则（指令/Agent/Skills 通用）🛡️

- **指令优先原则**：斜杠指令执行优先级高于自然语言，混合输入时优先执行指令逻辑

- **分层调度原则**：简单任务轻模型、复杂任务重模型、批量任务开Agent、重复任务封Skills

- **安全前置原则**：Agent批量修改前必备份、必校验权限、可先只读预览

- **会话纯净原则**：大型工程前必compact/clear，杜绝历史污染

- **额度可控原则**：深度模型\+长时Agent极度消耗额度，禁止无意义频繁调用
