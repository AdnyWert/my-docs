### 4\.2 企业级全局规则指令（可直接写入工程）

你可以将以下规则设置为项目永久规则，后续所有 Codex 操作自动遵守：

```Plain Text
你现在是本项目专属工程开发 AI，永久遵守以下项目规约：
1. 架构：模块化分层架构，严格区分入口层、业务层、工具层、配置层
```

### 4\.3 超大项目上下文解决方案✨

普通对话存在上下文上限，高级解决方案：**分层迭代 \+ 模块锁定 \+ 工程规则固化**

- 将大型项目拆分为：架构层、业务层、工具层、配置层

- 单次会话只迭代一个模块，避免上下文溢出

- 固定工程规则，AI 无需重复记忆规范

- 使用 CLI 局部扫描，精准修改，不全局污染

## 第7章 企业级安全、权限与性能调优🔐

### 7\.1 高级安全规范（必遵守）

- **禁止上传隐私文件**：密钥、证书、个人配置、隐私数据

- **目录白名单机制**：通过 \.codex 配置限制 AI 仅访问指定目录

- **代码脱敏**：批量替换明文密码、密钥、内网地址

- **危险操作校验**：删除、覆盖、批量修改前强制提示确认

### 7\.2 速率与性能调优

- 批量任务**分批次执行**，避免高频调用受限

- 大型项目采用**模块隔离迭代**，降低上下文压力

- 定期清理 `.codex/` 缓存，避免会话冗余卡顿

- 复杂任务使用 CLI 后台执行，不占用前端会话

## 第8章 高阶故障排查：顶级问题解决方案🧰

### 8\.1 CLI 常见高级报错与修复

- **报错**：版本不兼容
**方案**：锁定 Node 20LTS，重装全局 codex

- **报错**：权限不足无法读写本地文件
**方案**：开启目录权限、关闭只读、重新 codex init

- **报错**：上下文超限
**方案**：模块拆分、清空缓存、分层迭代

- **报错**：API 速率限制
**方案**：分批任务、增加间隔、优化单次任务负载

### 8\.2 大型项目迭代失控解决方案

- 固化工程规则，避免每次重新解释需求

- 单一会话只负责一个模块，杜绝多功能混杂

- 每次迭代输出**变更日志**，人工核对关键修改
